# 《龙虾系统》小说设计文档

> 创建日期：2026-03-02
> 状态：初版确认

## 一、基本信息

**暂定书名：** 龙虾系统（备选：数据化异界）

**类型：** 西方奇幻 + 穿越 + 轻松冒险

**篇幅：** 不设限，先写着看

## 二、核心设定

### 主角

- **身份：** 程序员，在用OpenClaw开发游戏时穿越
- **穿越后：** 复生到刚被哥布林灭团的佣兵小队，成为盗贼
- **金手指：** 「龙虾系统」——看世界全是数据
- **心态：** 佛系探索，不急着回家，不当牛马，能拯救世界就拯救，不能就游历全球

### 龙虾系统（大龙虾 🦞）

- **来源：** OpenClaw的外号叫龙虾，所以系统叫龙虾系统
- **定位：** 辅助AI，与主角协作
- **能力：**
  - 编写和优化战斗脚本
  - 分析数据、计算最优解
  - 提供技能建议
- **限制：**
  - 不直接控制身体，执行靠主角
  - 程序无法应对复杂临场情况

### 数据视角（分阶段解锁）

**初始能力：**
- 看到基础属性：血量、等级、职业
- 看到物品品质（灰/白/绿/蓝/紫/金）
- 看到简单概率（攻击命中率）

**进阶能力（解锁MCP模式后）：**
- 查看技能隐藏参数（前摇帧数、伤害系数）
- 分析战斗数据，优化输出循环
- 把复杂动作"录制"成可重复的宏

**高级能力（系统升级）：**
- 看到NPC真实好感度/隐藏动机
- 预判事件触发概率
- 调试魔法结构（找bug）

### MCP操作人体

- 主角发现身体可以被"编程"
- 可以写脚本：如`背刺连招.v1`
- 战斗时调用脚本，但不强制执行
- 人可以在关键时刻临场反应，超越程序

### 世界观

- **风格：** 西方奇幻底色
- **力量体系：** 混合（魔法+职业+符文/神器等多元）
- **特点：** 主角的数据视角能发现别人看不到的规律

## 三、开篇设计

### 第一章要点

1. 穿越瞬间（下班路上→哥布林尸体堆里醒来）
2. 发现龙虾系统（眼前弹出数据面板，大龙虾登场）
3. 危机（哥布林还在附近？或者刚离开？）
4. 发现幸存者（佣兵小队唯一活口，状态不好）
5. 决定：先活下来，再探索世界

### 幸存者角色

- 用来介绍世界背景
- 帮助主角融入社会
- 可能成为第一个伙伴

## 四、故事走向

**主线：** 探索世界 + 偶尔拯救世界

**风格：** 
- 轻松有趣
- 程序员梗
- 人机协作的化学反应
- 佛系但有原则
