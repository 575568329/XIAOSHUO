## 区域信息结构

`json
{
  "区域ID": {
    "name": "显示名称",
    "race": "种族",
    "blood": "血脉（可选）",
    "status": "状态",
    "danger": "危险等级",
    "resources": "主要资源",
    "monsters": "主要魔兽",
    "flight": "是否禁飞",
    "terrain": "地理环境",
    "climate": "气候",
    "cities": "主要城市（已开发区域）",
    "towns": "卫星城镇（已开发区域）",
    "desc": "简介（包含所有信息）"
  }
}
`

---

## 字段说明

### race（种族）
- human: 人类 🟦
- orc: 兽人 🔴
- elf: 精灵 🟩
- dwarf: 矮人 🟧
- dragon: 龙裔 💜
- mermaid: 海妖族 🧜
- byss: 深渊族 💜
- goblin: 地精 🟤
- undeveloped: 未开发 🌲
- 
one: 无人区 ❄️

### status（状态）
- 已开发: 城市区域
- 部分开发: 有城市但未完全开发
- 未开发: 无人区域
- 圣地: 精灵圣地
- 禁地: 深渊族领地
- 无人区: 极北冰原

### danger（危险等级）
- 低: 新手区域
- 低-中: 初级冒险区域
- 中: 中级冒险区域
- 中高: 高级冒险区域
- 高: 危险区域
- 极高: 死亡区域

### flight（禁飞）
- 允许: 可以自由飞行
- 限制: 需要特殊条件（迷雾、风暴等）
- 禁止: 完全禁止飞行

## 维护建议

1. **统一修改 regions.json**
   - 只修改一个文件
   - 定期同步到 index.html

2. **保持字段一致**
   - 所有区域使用相同字段结构
   - 缺失字段使用默认值

3. **版本控制**
   - 每次大改动创建备份
   - 使用 Git 管理（如果需要）

4. **测试流程**
   - 修改后刷新地图
   - 检查所有区域显示
   - 验证颜色和提示信息
